構造説明

Antigravityが認識するワークスペースのディレクトリ構造と、各ファイルの配置・役割について解説します。

推奨されるディレクトリ構造

Antigravityを導入したプロジェクトのルートディレクトリは、以下の構成を標準とします。

MyProject/
├── .antigravity_config/    // 配布用テンプレートフォルダ
├── .vscode/               // エディタ固有の設定
│   └── settings.json      // テレメトリ無効化など
├── src/                   // ソースコード本体
├── docs/                  // 設計書・仕様書
├── AGENTS.md              // エージェント定義（重要）
├── instructions.md        // AIへのタスク指示書
├── .env                   // 機密情報（AIから秘匿）
└── .gitignore             // AIの検索対象外設定
重要: ワークスペースの「ルート」をAntigravityに設定することで、AIはこれらすべてのファイルを相互に関連付けて解析できるようになります。

主要ファイルの役割

  • AGENTS.md
    プロジェクトのルートに配置。AIが「自分は何者か」「どのようなコーディング規約を守るべきか」を判断するための最優先ファイルです。
  • instructions.md
    AIに対する具体的な作業依頼を記述します。複雑なタスクを依頼する際、AIはこのファイルを読み込んで実装の拠り所とします。
  • .vscode/settings.json
    ワークスペース単位でのエディタ動作（テレメトリのオフ、自動保存の設定など）を定義します。
  • .env / .gitignore
    機密情報の分離と、AIに読み込ませたくないフォルダ（node_modules等）を除外するために適切に設定する必要があります。

設定の適用範囲

Antigravityの設定は、以下の2つのレベルで管理されます。

  • Global（グローバル）: エディタ全体の設定。日本語化ルール（User Rules）などはここで行います。
  • Workspace（ワークスペース）: プロジェクトごとの設定。AGENTS.md などのファイルベースの設定がこれにあたります。

基本的には「ワークスペース設定（AGENTS.md等）」が優先され、プロジェクトごとに最適なAIエージェントが構築されます。
