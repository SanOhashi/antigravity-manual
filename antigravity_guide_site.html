Antigravity 総合運用ガイド

なぜこのルールが必要なのか？背景にある技術的ロジックとベストプラクティスを解説します。

利用上の重大な注意点

なぜ機密情報を完全に分離すべきなのか

AIは文脈にあるコードを学習・参照するため、一度コード内にパスワードが混入すると、AIが新しいコードを生成する際にその機密情報を「正しい書き方」として再利用してしまうリスクがあります。

Technical Logic

.env への分離は、AIによる「機密情報の学習と再出力」の連鎖を物理的に断ち切るために不可欠な措置です。

セットアップの根拠

なぜ AGENTS.md が不可欠なのか

AIのデフォルト知識は一般的すぎて、プロジェクト固有の「暗黙の了解」を知りません。AGENTS.mdがない状態は、「自社のルールを全く知らない新人に、指示書なしで仕事を頼んでいる状態」と同じです。

Logic

AGENTS.mdによりAIの思考に強力な制約をかけます。これにより、レビューの手間を80%削減し、最初からプロジェクト基準に合致したコードを得られます。

開発フローの根拠

なぜ Planning Mode で計画を立てるのか

いきなりコードを書き始めると、AIは「全体的な整合性」を見失うことがあります。

System Logic

計画を立てることで、AIは修正前にすべての依存関係をスキャンし、不必要なトークン消費を抑え、最もシンプルで影響の少ない修正ルートを特定できます。
