Antigravity 総合運用ガイド

具体的な手順とその背景にある理由を統合した、実用的な運用マニュアルです。

利用上の重大な注意点

機密情報の漏洩リスクと完全分離の徹底

AIは文脈にあるコードを学習・参照するため、一度コード内にパスワードが混入すると、AIが新しいコードを生成する際にその情報を「正しいパターン」として再利用（再生成）してしまうリスクがあります。これを防ぐには .env への分離が不可欠であり、AIによる「機密情報の学習と再出力」の連鎖を物理的に断ち切る必要があります。

初期セットアップの手順と理由

1 インストーラーの実行

OSに合わせたインストーラーを実行し、「PATHに追加」を有効にしてください。PATHを通すことで、どのディレクトリからでもAIツールが起動可能になり、AIによるターミナル操作の代行がシステム的に正しく動作するようになります。

2 Antigravity起動後のワークスペース設定

AIがプロジェクト全体を正しく把握するため、以下の手順で作業フォルダを開きます。

  1. VS Codeの File > Open Folder... からプロジェクトのルートを開く
  2. Do you trust the authors... で [Yes] を選択
  3. サイドバーにフォルダ構成が表示されていることを確認

単一のファイル単位ではなくフォルダ全体で開くことで、AIは周辺ファイルや共通設定を文脈として読み込めるようになり、推論精度と整合性が劇的に向上します。

3 日本語化設定 (Japanese Pack 導入)

拡張機能から以下の手順で導入します。

  • 「Extensions」アイコンで Japanese Language Pack を検索して Install
  • 再起動して適用

UIだけでなくAIの内部的な思考プロセスを日本語に固定し、プロンプトの意図と回答の齟齬を最小化するために不可欠な設定です。

4 設定テンプレートの適用

.antigravity_config/ をルートに配置し、以下の命令をチャットに送信してプロジェクト固有のルールをAIに教え込みます。

フォルダ「.antigravity_config」内のファイルを現在のワークスペースに適用し、AGENTS.mdの必須ルールを反映してください。

AIはデフォルトで一般的な知識しか持たないため、この設定で「プロジェクト憲法」を与えます。これにより自社ルールを無視した不適切なコード生成を防ぎ、人間のレビュー工数を大幅に削減します。

開発フローの標準化

Planning Mode での計画策定

いきなりコードを修正せず、まず計画書（Implementation Plan）を確認し承認します。修正前に全依存関係をスキャンさせることで、デグレードを防ぎ、最小の修正ルートで品質を担保するためです。

instructions.md による指示の永続化

重要な要件はチャットではなく instructions.md に記述します。会話の長期化による要件の「聞き逃し」を防ぎ、AIが常に最新の仕様書を直視している状態を維持するためです。
